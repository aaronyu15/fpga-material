# Installation Steps
>[!Tip] Before you begin
> Refer to [UG973 Installation and setup](https://docs.amd.com/r/en-US/ug973-vivado-release-notes-install-license/Release-Notes)  for the full installation procedure.
> Install Vivado and Vitis by downloading the installer from the main [download page](https://www.xilinx.com/support/download.html).
> Make sure the operating system being used is [supported](https://docs.amd.com/r/en-US/ug973-vivado-release-notes-install-license/Supported-Operating-Systems) (Note to change the document to the version of Vivado you want to install).
> This will install Vivado and Vitis themselves as well as any selected board files. Drivers need to be installed separately on Linux.

Apparently newer Ubuntu versions have the USB-to-UART driver already (CP210x). To check if they exist, run 
```bash
modinfo usbserial 
modinfo cp210x 
```

You will still have to install these Vivado specific "Cable Drivers" (and allow group access to /dev/ttyUSB# if using UART). These commands perform both steps:
 ```bash
 < Vivado Install >/data/xicom/cable_drivers/lin64/install_script/install_drivers/
 sudo ./install_drivers
 sudo adduser $USER dialout
 ```

# Vivado
Vivado is the main tool that allows you to convert a hardware design (either from hand-coded RTL files or IP blocks) into a bitstream to program the Programmable Logic (PL) side of the FPGA. In Vivado you can perform simulation, synthesis, implementation, and bitstream programming. Aside from simulation, these steps depend on the board that is selected since the board determines the available hardware resources, pins, pinout layout, etc.

## Vivado User Guides and Tutorials

# Vitis
Vitis is a Vivado companion tool that has two main functions: to program embedded designs and to perform High Level Synthesis (HLS). For embedded designs, if the FPGA board contains hard logic, like the Zynq series which have various "CPU"s, then Vitis can be used to program them. This relates to the Processing System (PS) side of the FPGA. For HLS, Vitis can be used to convert C/C++/other code to a hardware implementation. Essentially, it translates high level algorithms it into RTL. This relates to the Programming Logic (PL) side of the FPGA.

>[!info]
> In addition to the official [Design Hubs Landing Page](https://docs.amd.com/p/design-hubs), Vitis has its own landing page [UG1416](https://docs.amd.com/v/u/en-US/ug1416-vitis-documentation). The main Design Hubs Landing Page has links to the Vitis user guides, but does not seem to link to the tutorials. This is the main difference, aside from a pretty interface.
