    Apr 08, 202611 min read

    A Practical Guide to Tcl Scripting: Syntax and Templates Tcl (Tool Command Language) is a powerful, dynamic scripting language known for its simple syntax and easy integration with other applications, especially in the domain of electronic design automation (EDA) and testing. This guide provides a collection of the most common syntax patterns and templates.

    1. Tcl Fundamentals: The “Everything is a Command” Principle In Tcl, every statement is a command, including variable assignments, loops, and procedure definitions. A command is a list of words separated by spaces, where the first word is the command name and subsequent words are its arguments.

    Basic Script Structure

    Variables and Substitution set: Assigns a value to a variable. Quotes are optional unless there are spaces/special characters. $: Substitutes the value of a variable. []: Command Substitution. Executes the command inside the brackets and substitutes its result. {}: Groups words into a single argument without performing substitutions. Treats the content as a literal string. "": Groups words into a single argument with substitutions.

    Automatically created variables in a script.

    • $argc - Number of command-line arguments passed to the script.
    • $argv - List of command-line arguments.

    • $env - Array containing environment variables. Access like $env(PATH, $env(HOME)
    • $auto_path - list of directories to search for packages

    Other Notes

    • User defined namespaces can be used to avoid conflicting the global space
    • Weird ass notation for scope names with ::. The first :: refers to the global namespace, so you can also write namespace eval ::reporting to define the namespace relative to the global scope, instead of the possible sub-scope when it is “imported”.
    • Importing packages (packages are not the same as user defined namespaces)

    List of common commands

    This guide provides a detailed list of common Tcl commands whose primary purpose is to return a value, making them ideal candidates for use within [] command substitution blocks.

    1. List Manipulation

    These commands are the workhorses for handling ordered collections of data.

    CommandSyntax & PurposeExample & Return Value
    llengthllength list
    Returns the number of elements in a list.    		set n [llength {a b c}]
    Returns: 3
    lindexlindex list index
    Returns the element at a specific index.    		set item [lindex {a b c} 1]
    Returns: b
    lrangelrange list first last
    Returns a new list containing a range of elements.    		set sub [lrange {a b c d} 1 2]
    Returns: b c
    lsearchlsearch ?options? list pattern
    Finds an element matching a pattern and returns its index, or -1 if not found.    		set idx [lsearch {a b c} b]
    Returns: 1
    lsortlsort ?options? list
    Returns a new list with the elements sorted.    		set sorted [lsort -integer {5 1 3}]
    Returns: 1 3 5
    joinjoin list ?joinString?
    Joins the elements of a list into a single string.    		set str [join {a b c} ","]
    Returns: a,b,c
    splitsplit string ?splitChars?
    Splits a string into a list.    		set list [split "a,b,c" ","]
    Returns: a b c
    listlist ?arg1 arg2 ...?
    Creates a properly escaped list from its arguments.    		set list [list "An item" "Another"]
    Returns: {An item} Another

    2. String Manipulation

    Essential for parsing and formatting text.

    CommandSyntax & PurposeExample & Return Value
    stringstring sub-command string ?args?
    A suite of commands for string operations.
      lengthstring length stringset len [string length "Tcl"]
    Returns: 3
      rangestring range string first lastset sub [string range "abcde" 1 3]
    Returns: bcd
      mapstring map {charMap} string
    Replaces multiple characters in one pass.    		set new [string map {a 1 b 2} "abc"]
    Returns: 12c
      trimstring trim string ?chars?
    Removes leading/trailing characters (whitespace by default).    		set clean [string trim " hi "]
    Returns: hi
      toupperstring toupper string
    Make an uppercase string.    		set var [string toupper "hi"]
    Returns: HI
    formatformat formatString ?val...?
    Creates a formatted string like C’s sprintf.    		set hex [format "0x%04X" 255]
    Returns: 0x00FF
    scanscan string format ?var...?
    Parses a string according to a format and populates variables. Returns the number of successful conversions.    		set n [scan "id:123" "id:%d" my_id]
    Returns: 1 (and sets my_id to 123)
    regexpregexp ?opts? exp str ?match...?
    Matches a regular expression. Returns 1 on success, 0 on failure. Populates match variables.    		set found [regexp {(\d+)} "age:42" -> num]
    Returns: 1 (and sets num to 42)

    3. File System Interaction

    For querying and manipulating files and directories.

    CommandSyntax & PurposeExample & Return Value
    filefile sub-command name ?args?
    A suite of commands for file attributes.
      existsfile exists nameset found [file exists data.txt]
    Returns: 1 or 0
      sizefile size nameset bytes [file size data.txt]
    Returns: The file size in bytes.
      isdirfile isdirectory nameset isdir [file isdirectory /etc]
    Returns: 1 or 0
      joinfile join path1 path2 ...
    Joins path components with the correct separator.    		set p [file join /home user file.txt]
    Returns: /home/user/file.txt
      dirnamefile dirname pathset dir [file dirname /a/b/c.txt]
    Returns: /a/b
      tailfile tail pathset name [file tail /a/b/c.txt]
    Returns: c.txt
    globglob ?switches? pattern
    Returns a list of files matching a wildcard pattern.    		set files [glob *.tcl]
    Returns: A list of Tcl files.
    pwdpwd
    Returns the current working directory.    		set cwd [pwd]
    Returns: The full path of the current directory.
    openopen fileName ?access?
    Opens a file. Returns a channel handle (e.g., file3).    		set chan [open data.txt r]
    Returns: A channel ID like file3.
    getsgets channelId ?varName?
    Reads a line from a channel. Returns the number of characters read, or a negative number on EOF. If varName is given, the line is stored there.    		set num_chars [gets $chan line]
    Returns: Number of chars read.

    4. Introspection and State

    For querying the state of the Tcl interpreter itself.

    CommandSyntax & PurposeExample & Return Value
    infoinfo sub-command ?args?
    Provides information about the Tcl state.
      existsinfo exists varNameset v_exists [info exists my_var]
    Returns: 1 or 0
      commandsinfo commands ?pattern?set list_cmds [info commands l*]
    Returns: list llength lindex lsort ...
      procsinfo procs ?pattern?set my_procs [info procs]
    Returns: A list of all defined procedures.
      levelinfo level ?number?set stack [info level 0]
    Returns: The command at the top of the call stack.
      scriptinfo scriptset script_path [info script]
    Returns: The path to the currently executing script.

    5. Execution and Flow Control

    CommandSyntax & PurposeExample & Return Value
    exprexpr {expression}
    Evaluates a mathematical or logical expression.    		set val [expr { (5 * 4) + 2 }]
    Returns: 22
    catchcatch {script} ?resultVar?
    Executes a script and traps errors. Returns 0 on success, 1 on error.    		set status [catch {set x [expr {1/0}]} msg]
    Returns: 1 (and sets msg to “divide by zero”)
    execexec ?switches? arg1 ?arg2 ...?
    Executes an external command. Returns the standard output of that command.    		set who [exec whoami]
    Returns: The current username.
    incrincr varName ?increment?
    Increment varName by a specific amount. Default adds 1.    		set var [incr var 1]

    6. Date and Time

    CommandSyntax & PurposeExample & Return Value
    clockclock sub-command ?args?
    Suite of commands for time operations.
      secondsclock secondsset epoch [clock seconds]
    Returns: Integer seconds since 1970-01-01.
      formatclock format seconds ?opts?set date [clock format $epoch -format %Y-%m-%d]
    Returns: 2025-06-29
      scanclock scan dateString ?opts?set epoch [clock scan "2025-06-29"]
    Returns: The epoch time for that date.
